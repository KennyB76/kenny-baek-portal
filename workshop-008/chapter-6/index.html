Section A
RDB — 관계형 데이터베이스

관계형 데이터베이스(RDB)는 데이터를 표(Table) 형태로 저장하고, 테이블 간 관계(Relation)로 연결한다. 각 행(Row)은 레코드, 각 열(Column)은 필드다. Primary Key(PK)로 각 행을 고유 식별하고, Foreign Key(FK)로 다른 테이블과 연결한다. PostgreSQL, MySQL, SQLite가 대표적이다.

Section B
SQL 4동사 (CRUD)

SQL(Structured Query Language)은 RDB와 대화하는 언어다. 핵심 4동사는 CRUD 작업에 대응한다.

SELECT * FROM students WHERE grade = 'A'; -- Read INSERT INTO students (name, grade) VALUES ('Trinity', 'A'); -- Create UPDATE students SET grade = 'A+' WHERE name = 'Trinity'; -- Update DELETE FROM students WHERE name = 'Trinity'; -- Delete

Section C
NoSQL

NoSQL(Not Only SQL)은 RDB의 표 구조를 벗어나 유연한 데이터 모델을 사용한다. 스키마 없이 빠른 개발이 가능하고 대규모 수평 확장에 유리하다. 단, 복잡한 JOIN 쿼리가 어렵고 트랜잭션 지원이 제한적일 수 있다.

Section D
인덱스 / 정규화

인덱스(Index)는 책의 색인처럼 특정 컬럼 기준으로 빠르게 검색할 수 있게 한다. 읽기 속도는 향상되지만 쓰기 속도와 저장 공간을 소비한다. 정규화(Normalization)는 데이터 중복을 제거해 일관성을 높이는 설계 원칙이다. 1NF → 2NF → 3NF 단계로 적용한다.

개념효과주의점
인덱스 (Index)SELECT 속도 대폭 향상INSERT/UPDATE 느려짐, 공간 소비
정규화 (Normalization)중복 제거, 데이터 일관성JOIN 증가로 쿼리 복잡도 상승
비정규화 (Denormalization)JOIN 줄여 읽기 속도 향상중복 데이터 관리 필요
Section E
ORM / 트랜잭션

ORM(Object-Relational Mapping)은 코드의 객체(Class)와 DB 테이블을 자동으로 매핑한다. SQL을 직접 쓰지 않고 객체 메서드로 DB 조작이 가능하다. 트랜잭션(Transaction)은 여러 DB 작업을 하나의 단위로 묶는다. 모두 성공하거나(Commit), 하나라도 실패하면 모두 취소(Rollback)한다. ACID(Atomicity, Consistency, Isolation, Durability)가 트랜잭션의 4대 속성이다.

Quiz
  1. 관계형 DB에서 Primary Key와 Foreign Key의 역할 차이를 설명하고, JOIN이 필요한 이유를 말하라.
  2. SQL 4동사(SELECT/INSERT/UPDATE/DELETE)를 HTTP 메서드(GET/POST/PUT/DELETE)와 각각 대응시켜라.
  3. Document DB(MongoDB)와 RDB(PostgreSQL)를 사용하기에 각각 적합한 상황을 예시와 함께 설명하라.
  4. 인덱스를 추가하면 읽기 속도가 향상되는 이유와 쓰기 속도가 느려지는 이유를 설명하라.
  5. 트랜잭션의 ACID 속성 중 Atomicity와 Isolation을 설명하고, 실생활 예시를 들어라.
