# Chapter 6 핵심 용어

| 용어 | 정의 | Kenny 비유 |
|------|------|-----------|
| 데이터베이스 | 데이터 저장/관리 소프트웨어 | 도서관 |
| RDB | 테이블 구조 관계형 DB. MySQL/PostgreSQL | 엑셀 장부 |
| NoSQL | 비정형 데이터 저장. MongoDB/Redis | 포스트잇 메모 |
| 테이블 | 행+열로 구성된 데이터 단위 | 엑셀 시트 |
| 레코드 (행) | 테이블의 가로 줄 하나 = 데이터 1개 | 엑셀 한 줄 |
| 컬럼 (열) | 테이블의 세로 항목 (이름/나이 등) | 엑셀 제목 행 |
| Primary Key | 레코드 고유 식별값. 중복 불가 | 학생 학번 |
| Foreign Key | 다른 테이블 PK 참조. 관계 연결 | 성적표에 학번 적기 |
| SQL | Structured Query Language. DB 조작 언어 | DB 와 소통하는 언어 |
| SELECT | 데이터 조회 | 성적 검색 |
| INSERT | 새 데이터 추가 | 신입생 등록 |
| UPDATE | 기존 데이터 수정 | 주소 변경 |
| DELETE | 데이터 삭제 | 졸업 후 삭제 |
| JOIN | 여러 테이블 합쳐 조회 | 학생+성적+수업 합산 |
| 인덱스 | 검색 속도 향상용 주소록. 공간 비용 | 책 목차 |
| 트랜잭션 | 여러 쿼리를 하나로 묶음. 전체 성공/실패 | 계좌 이체 (동시 출금+입금) |
| ACID | 원자성/일관성/격리성/지속성. 트랜잭션 4원칙 | 거래 신뢰 보장 |
| 정규화 | 중복 최소화 테이블 분리 설계 | 카테고리별 폴더 정리 |
| ORM | 코드 객체 ↔ DB 테이블 자동 변환 | 번역기 |
| Redis | 메모리 기반 초고속 Key-Value 저장소 | 포스트잇 (빠른 메모) |
