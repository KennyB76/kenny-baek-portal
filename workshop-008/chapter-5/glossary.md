# Chapter 5 핵심 용어

| 용어 | 정의 | Kenny 비유 |
|------|------|-----------|
| 클라이언트 | 서버에 요청 보내는 쪽 | 손님 (주문하는 사람) |
| 서버 | 요청 받아 응답 주는 쪽 | 주방 (요리해주는 곳) |
| IP 주소 | 인터넷 상 장치 고유 주소 (192.168.x.x) | 집 주소 |
| 도메인 | IP 대신 쓰는 사람 읽기 좋은 주소 | 상호명 |
| DNS | 도메인 → IP 변환 서버 | 전화번호부 |
| HTTP | HyperText Transfer Protocol. 웹 통신 규약 | 편지 쓰는 형식 |
| HTTPS | HTTP + TLS 암호화 | 봉투에 넣은 편지 (암호화) |
| 포트 | IP 에서 서비스 구분 번호 | 집 내부 방 번호 |
| TCP | 전송 제어 프로토콜. 순서 보장 + 재전송 | 등기 우편 (서명 확인) |
| UDP | 순서 보장 없는 빠른 전송 | 일반 우편 (빠르지만 분실 가능) |
| 패킷 | 데이터를 쪼갠 전송 단위 | 택배 상자 (분할 배송) |
| 라우터 | 패킷 경로 결정 장치 | 우체국 분류 센터 |
| 방화벽 | 허가된 트래픽만 통과 | 경비원 |
| NAT | 사설 IP ↔ 공인 IP 변환 | 회사 대표 전화 (내부 번호 숨김) |
| VPN | 암호화 터널 통신 | 비밀 통로 |
| Tailscale | WireGuard 기반 P2P VPN | 직통 비밀 전화 |
| CDN | 전 세계 엣지 서버에 콘텐츠 복사 | 전국 지점 (가까운 곳에서 배달) |
| 로드밸런서 | 서버 여러 대로 트래픽 분산 | 계산대 여러 개 (줄 분산) |
| 웹 서버 | 정적 파일(HTML/CSS/JS) 서빙 | 셀프 자판기 |
| WAS | 동적 비즈니스 로직 처리 서버 | 직원이 직접 만들어주는 곳 |
