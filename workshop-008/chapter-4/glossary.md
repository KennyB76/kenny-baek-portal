# Chapter 4 핵심 용어

| 용어 | 정의 | Kenny 비유 |
|------|------|-----------|
| CPU | 계산 전담 프로세서. 컴퓨터의 뇌 | 선생님 (계산하는 사람) |
| 코어 | CPU 내 독립 처리 단위. 멀티코어 = 병렬 처리 | 선생님 수 |
| 클럭 속도 | CPU 연산 속도 (GHz). 높을수록 빠름 | 선생님 말하는 속도 |
| RAM | 실행 중 임시 저장. 전원 끄면 삭제 | 책상 위 (작업 공간) |
| 디스크 (SSD) | 반영구 파일 저장. 전원 꺼져도 유지 | 책장 (영구 보관) |
| 캐시 메모리 | CPU 내 초고속 소용량 메모리 | 손에 들고 있는 메모 |
| 가상 메모리 | 디스크를 RAM 처럼 쓰는 OS 트릭 | 바닥에 임시로 쌓아둔 책 |
| OS | 하드웨어 ↔ 소프트웨어 중개 소프트웨어 | 교장 선생님 (자원 배분) |
| 프로세스 | 실행 중인 프로그램 = 파일이 메모리에 올라온 상태 | 수업 중인 학생 |
| 쓰레드 | 프로세스 내 실행 단위. 메모리 공유 | 수업 중 모둠 활동 |
| 시분할 | CPU 시간을 프로세스별로 쪼개 빠르게 번갈아 실행 | 선생님이 학생 한 명씩 순서대로 봐줌 |
| CPU 스케줄링 | 어느 프로세스에 CPU 시간 얼마나 배분할지 결정 | 숙제 우선순위 결정 |
| 인터럽트 | 외부 이벤트로 CPU 현재 작업 중단 | 수업 중 전화 (긴급 상황) |
| 교착상태 | 두 프로세스가 서로 자원 기다리며 멈춤 | 교통 사거리 교착 |
| 가상화 | 물리 서버를 여러 가상 서버로 분할 | 교실 칸막이 (공간 분리) |
| VM | Virtual Machine. 가상 컴퓨터 | 칸막이 교실 |
| Docker | 컨테이너 기반 경량 가상화 | 간이 칸막이 (VM 보다 가벼움) |
| 파일시스템 | 디스크 파일 구조 관리 (FAT/NTFS/ext4) | 책장 분류 체계 |
| 드라이버 | OS ↔ 하드웨어 연결 소프트웨어 | 번역가 (하드웨어 언어 통역) |
| 부팅 | 전원 켜고 OS 로딩 과정 | 출근 준비 |
