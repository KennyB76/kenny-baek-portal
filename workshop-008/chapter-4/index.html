Section A
폰노이만 구조

현대 컴퓨터는 폰노이만(Von Neumann) 구조를 기반으로 한다. 프로그램(명령어)과 데이터를 같은 메모리에 저장하고 CPU가 순서대로 읽어 실행하는 방식이다. CPU, 메모리(RAM), 저장장치(SSD/HDD), 입출력 장치(I/O)가 버스(Bus)로 연결된다.

입력 → CPU (ALU + 제어장치 + 레지스터) ↑↓ 버스(Bus) RAM (프로그램 + 데이터 공존) ↑↓ SSD/HDD (영구 저장) → 출력
Section B
메모리 계층

컴퓨터 메모리는 속도와 용량이 반비례하는 계층 구조를 갖는다. 빠를수록 비싸고 작으며, 느릴수록 용량이 크다. CPU는 레지스터 → 캐시 → RAM 순으로 데이터를 찾고, 없으면 디스크에서 로드한다.

Section C
OS 역할

운영체제(OS)는 하드웨어와 응용 프로그램 사이의 중간 계층이다. 프로세스 관리, 메모리 관리, 파일 시스템, 네트워크 스택, 보안을 담당한다. Windows, macOS, Linux(서버 95% 점유)가 대표적이다.

Section D
프로세스 / 쓰레드

프로세스(Process)는 실행 중인 프로그램의 독립적 인스턴스다. 독립된 메모리 공간을 가진다. 쓰레드(Thread)는 프로세스 안의 실행 단위로, 같은 메모리 공간을 공유하며 더 가볍다. 멀티프로세스는 안정적이나 무겁고, 멀티쓰레드는 빠르나 공유 메모리 충돌(Race Condition)에 주의가 필요하다.

Section E
스케줄링 / 시분할

현대 OS는 CPU가 하나여도 여러 프로세스를 동시에 실행하는 것처럼 보이게 한다. 시분할(Time-sharing)로 각 프로세스에 CPU 시간을 빠르게 번갈아 할당하기 때문이다. 스케줄링 알고리즘(FCFS, Round Robin, Priority 등)이 순서를 결정한다.

Quiz
  1. 폰노이만 구조의 핵심 특징을 한 문장으로 설명하고, 현대 컴퓨터에서 이 구조의 병목(Von Neumann Bottleneck)이 왜 발생하는지 말하라.
  2. 레지스터, 캐시, RAM, SSD의 속도와 용량 차이를 계층 순서로 정리하라.
  3. OS가 프로세스 관리에서 하는 역할 3가지를 열거하라.
  4. 프로세스와 쓰레드의 메모리 공유 방식 차이를 설명하고, Race Condition이 무엇인지 말하라.
  5. 시분할 방식이 없다면 싱글코어 CPU에서 여러 앱을 동시에 실행하는 것이 가능한가? 그 이유를 설명하라.
