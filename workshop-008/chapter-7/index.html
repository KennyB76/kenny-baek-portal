Section A
아키텍처 = 설계도

IT 아키텍처는 시스템을 구성하는 컴포넌트와 그 관계를 정의하는 설계도다. 좋은 아키텍처는 가용성, 확장성, 유지보수성, 보안을 동시에 고려한다. 아키텍처는 비즈니스 요구사항에서 출발하며, 정답이 하나가 아니라 트레이드오프의 선택이다.

관심사설명
가용성 (Availability)시스템이 얼마나 오래 정상 작동하는가 (9999 = 99.99%)
확장성 (Scalability)트래픽 증가 시 성능 유지 능력 (수직 vs 수평 확장)
유지보수성 (Maintainability)코드/시스템 변경이 얼마나 쉬운가
보안 (Security)인증, 인가, 암호화, 최소 권한 원칙
Section B
3-Tier 아키텍처

3-Tier는 웹 서비스의 전통적 구조다. 프레젠테이션(FE), 비즈니스 로직(BE), 데이터(DB) 3층으로 분리한다. 각 층은 독립적으로 확장·교체 가능하며, 역할 분리로 유지보수성이 높다.

[Tier 1] 프레젠테이션 브라우저 / 앱 (React, Next.js) ↑↓ HTTP/API [Tier 2] 비즈니스 로직 서버 (Node.js, Django, Spring) ↑↓ SQL / ORM [Tier 3] 데이터 DB 서버 (PostgreSQL, Redis)
Tier역할NewtonHQ 대응
Tier 1 — FEUI, 사용자 인터랙션Kenny (요청 주체)
Tier 2 — BE비즈니스 로직, APIjarvis (실행 + 판단)
Tier 3 — DB데이터 영구 저장memory/ 카드 파일 시스템
Section C
모놀리스 → MSA 진화

초기 서비스는 대부분 모놀리식으로 시작한다. 규모가 커지면 한 팀이 배포하면 다른 기능이 영향을 받고, 특정 기능만 스케일 아웃이 어렵다. MSA는 각 기능을 독립적인 서비스로 분리해 이 문제를 해결한다. 단, 서비스 간 통신 복잡도와 분산 트랜잭션 관리가 추가된다.

진화 단계특징적합 시점
Monolith단일 코드베이스, 빠른 초기 개발MVP, 소규모 팀
Modular Monolith내부 모듈 분리, 단일 배포중간 규모
MSA독립 서비스 + 독립 DB대규모, 다팀 운영
Section D
로드밸런서 / 스케일링

로드밸런서(Load Balancer)는 들어오는 트래픽을 여러 서버에 분산한다. 단일 서버 과부하 방지 + 고가용성 확보. 수직 확장(Scale Up)은 서버 사양을 높이고, 수평 확장(Scale Out)은 서버 수를 늘린다. 클라우드 시대엔 수평 확장이 일반적이다.

개념설명NewtonHQ 대응
Load Balancer트래픽 분산 (Round Robin / 최소 연결)orchestrator/ 작업 분배
Scale Up서버 CPU/RAM 증설Claude 더 큰 모델 업그레이드
Scale Out서버 대수 증가sub-agent 병렬 확장
Auto Scaling트래픽 따라 자동 증감run_in_background 동적 실행
Section E
API Gateway / BFF

API Gateway는 클라이언트와 MSA 서비스 사이에서 단일 진입점(Single Entry Point) 역할을 한다. 인증, 라우팅, 속도 제한, 로깅을 일괄 처리한다. BFF(Backend For Frontend)는 클라이언트 유형(웹/모바일)별로 최적화된 API를 제공하는 패턴이다.

IT 개념NewtonHQ 대응
3-Tier 아키텍처Kenny / jarvis / memory/
API Gatewayorchestrator/ (단일 위임 진입점)
MSA 서비스도메인별 독립 sub-agent
로드밸런서orchestrator/ 작업 분배 룰
서비스 메시sub-agent 간 통신 프로토콜
Quiz
  1. IT 아키텍처 설계에서 가용성, 확장성, 유지보수성, 보안 중 트레이드오프가 발생하는 예를 하나 들어라.
  2. 3-Tier 아키텍처에서 각 Tier의 역할과 분리의 장점을 설명하라.
  3. 스타트업이 Monolith로 시작해 MSA로 전환하는 시점과 이유를 설명하라.
  4. Scale Up과 Scale Out의 차이를 설명하고, 클라우드 환경에서 Scale Out이 선호되는 이유를 말하라.
  5. API Gateway가 없을 때 MSA 클라이언트가 겪는 문제를 설명하고, API Gateway가 어떻게 해결하는지 말하라.
