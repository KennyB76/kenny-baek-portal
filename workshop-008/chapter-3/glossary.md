# Chapter 3 핵심 용어

| 용어 | 정의 | Kenny 비유 |
|------|------|-----------|
| 폭포수 (Waterfall) | 기획→분석→설계→개발→테스트 단방향 흐름 | 건물 짓기 (한 번 완성 후 납품) |
| Agile | 짧은 스프린트 반복 개발 방법론 | 리모델링 (조금씩 계속 개선) |
| Sprint | Agile 2-4주 단위 개발 사이클 | 한 학기 단위 수업 |
| 백로그 (Backlog) | 구현 대기 기능 목록 | 수업 준비 해야 할 것 목록 |
| 스크럼 (Scrum) | Agile 실행 프레임워크 | 팀 미팅 방식 |
| 데일리 스크럼 | 매일 아침 5-15분 진행 상황 공유 | 조회 시간 |
| 스토리 포인트 | 기능 구현 난이도/시간 추정 단위 | 수업 난이도 별점 |
| MVP | Minimum Viable Product. 최소 기능 버전 | 시험용 미니 버전 |
| CI | Continuous Integration. 코드 merge 시 자동 빌드+테스트 | 매일 숙제 검사 |
| CD | Continuous Deployment. 자동 배포 | 검사 통과 즉시 게시 |
| DevOps | 개발+운영 융합 문화 + 자동화 도구 | 선생+행정 겸임 |
| 모놀리스 | 모든 기능이 단일 코드베이스 | 올인원 교재 (한 권) |
| MSA | 기능별 독립 서비스 분리 | 과목별 교재 분리 |
| 리팩토링 | 기능 변경 없이 코드 구조 개선 | 노트 정리 (내용 동일) |
| 코드 리뷰 | 팀원이 서로 코드 검토 | 상호 채점 |
| 기술 부채 | 빠른 개발 위해 나쁜 코드 선택. 나중에 대가 지불 | 벼락치기 (나중에 복습 필요) |
| 배포 (Deploy) | 코드를 실제 서비스에 올리기 | 숙제 제출 |
| 롤백 | 문제 발생 시 이전 버전으로 복귀 | 제출 취소 |
| 스테이징 | 배포 전 실제 환경 테스트용 서버 | 시험 전 모의고사 |
| 카나리 배포 | 일부 사용자에게만 새 버전 적용 후 확대 | 일부 반 먼저 시험 후 전체 |
