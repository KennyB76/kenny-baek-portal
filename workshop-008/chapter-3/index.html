Section A
폭포수 vs Agile

폭포수(Waterfall)는 요구사항 분석 → 설계 → 개발 → 테스트 → 배포를 순서대로 진행한다. 각 단계가 완료되어야 다음으로 넘어가므로 변경이 어렵고 납기가 수개월~수년이다. Agile은 짧은 반복 주기(Sprint)로 작동 가능한 소프트웨어를 반복 배포하며 고객 피드백을 빠르게 반영한다.

Waterfall 요구사항 → 설계 → 개발 → 테스트 → 배포 (한 방향, 수개월) Agile [Sprint 1: 2주] → [Sprint 2: 2주] → [Sprint 3: 2주] → ... 요구 → 개발 → 테스트 → 배포 (매 Sprint 반복)
항목폭포수Agile
주기수개월~수년2~4주 Sprint
변경 대응어려움 (계획 고정)용이 (백로그 조정)
고객 참여초기 + 납품 시매 Sprint Review
위험후반부 발견초반 조기 발견
Section B
스프린트 / 스크럼

스크럼(Scrum)은 Agile 프레임워크 중 가장 널리 쓰이는 방식이다. Product Backlog(요구사항 목록)에서 Sprint Backlog를 뽑아 2주 Sprint를 실행한다. 매일 Daily Standup(15분)으로 진행 상황을 공유하고, Sprint 종료 시 Review + Retrospective를 진행한다.

Product Backlog (우선순위 정렬 요구사항 목록) | v Sprint Planning Sprint Backlog (이번 Sprint 할 일) | v Daily Standup x 10일 Sprint Review (작동 소프트웨어 시연) | v Retrospective (팀 프로세스 개선)
역할책임NewtonHQ 대응
Product Owner백로그 우선순위 결정Kenny (요구사항 결정)
Scrum Master장애 제거, 프로세스 촉진jarvis (워크플로 조율)
Dev TeamSprint 실행sub-agent 팀
Section C
CI/CD 파이프라인

CI(Continuous Integration)는 코드 변경 시 자동으로 빌드 + 테스트를 실행한다. CD(Continuous Delivery/Deployment)는 테스트 통과 시 자동 배포까지 진행한다. GitHub Actions, Jenkins, GitLab CI 등이 대표적인 도구다.

Code Push → git push origin main → CI 트리거: 자동 빌드 + 테스트 실행 → 테스트 PASS → CD 트리거: 스테이징 / 프로덕션 배포 → 테스트 FAIL → PR 블로킹 + 알림
단계동작NewtonHQ 예시
CI — Buildnpm run build / pytestGitHub Actions 자동 빌드
CI — Test단위 테스트 / 통합 테스트lint + HTML 검증
CD — Deploy스테이징 → 프로덕션 자동 반영GitHub Pages 자동 배포
Section D
모놀리스 vs MSA

모놀리식(Monolithic) 아키텍처는 모든 기능이 하나의 코드베이스에 묶여 있다. MSA(Microservices Architecture)는 각 기능을 독립적인 서비스로 분리해 배포·확장·장애 격리가 용이하다. Netflix, Uber 등 대규모 서비스는 MSA를 사용하지만 소규모 프로젝트는 모놀리스가 더 효율적일 수 있다.

항목모놀리스MSA
배포 단위전체 앱 한 번에서비스별 독립 배포
확장전체 스케일 아웃병목 서비스만 스케일 아웃
장애 격리한 곳 문제 → 전체 영향해당 서비스만 영향
복잡도낮음 (단순 구조)높음 (서비스 간 통신 필요)
NewtonHQ 대응초기 단일 포탈도메인별 sub-agent 분리
Section E
DevOps 문화

DevOps는 개발(Dev)과 운영(Ops)의 사일로를 허물고 협업·자동화·지속적 개선을 추구하는 문화다. 코드 배포 주기 단축, 장애 복구 시간 단소, MTTR(Mean Time To Recovery) 개선이 핵심 지표다. DevOps = 툴 도입이 아니라 팀 문화·프로세스의 변화다.

IT 개념NewtonHQ 대응
백로그_plan/ 폴더 (Kenny 전용 계획)
스프린트세션 단위 작업 사이클 (Kenny ↔ jarvis)
CI/CDGitHub Pages 자동 배포 파이프라인
Daily Standup세션 시작 시 communication.md 확인
MSA 서비스도메인별 독립 sub-agent
Retrospectivememory 카드 피드백 루프
