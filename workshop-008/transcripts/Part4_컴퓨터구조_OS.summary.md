# Part4 - 컴퓨터 구조 / OS

## 5 손가락 요약

1. 폰노이만 구조 - CPU + 메모리 + 디스크 = 지금도 유효. 60년 불변 골격.
2. 메모리 계층 - 캐시(CPU 내부·초고속) → 메인 메모리(16GB 기준) → 디스크(1TB). 속도↑ = 가격↑ = 용량↓.
3. OS 역할 - 하드웨어 추상화 + CPU/메모리 스케줄링. 프로그램이 하드웨어 직접 건드리지 않게 중개.
4. 프로세스 vs 쓰레드 - 프로세스 = 실행 중인 프로그램. 쓰레드 = 프로세스 내 실행 단위. 멀티쓰레드 = 같은 메모리 공유.
5. 시분할 (Time Sharing) - CPU 는 하나지만 여러 프로세스를 빠르게 번갈아 실행. 동시처럼 보이는 착시.

## 주요 용어 + 정의

| 용어 | 정의 |
|------|------|
| CPU | 계산 전담. 컴퓨터의 뇌. 코어 수 = 실제 병렬 처리 수 |
| 메모리 (RAM) | 임시 고속 저장. 프로그램 실행 중 데이터 올라오는 곳 |
| 디스크 (SSD/HDD) | 반영구 저장. 파일 저장 공간. 전원 꺼져도 유지 |
| 캐시 메모리 | CPU 내부 초고속 소용량 메모리. 자주 쓰는 명령어 선적재 |
| 가상 메모리 | 디스크를 메모리처럼 쓰는 트릭. 메모리 부족 시 OS 자동 사용 |
| OS | Operating System. 하드웨어 ↔ 소프트웨어 중개. Windows/macOS/Linux |
| 프로세스 | 실행 중인 프로그램. 파일 → 메모리 로드 → 실행 상태 |
| 쓰레드 | 프로세스 내 실행 단위. 같은 메모리 공유. 가볍고 빠름 |
| 시분할 | CPU 시간을 프로세스별로 쪼개 배분. 동시 실행처럼 보이게 함 |
| CPU 스케줄링 | 어느 프로세스에 CPU 시간 얼마나 줄지 결정하는 OS 알고리즘 |
