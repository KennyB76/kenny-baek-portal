Group A

API = 약속된 통신 계약

API (Application Programming Interface) 는 시스템 간 데이터를 교환하는 규약. 웨이터 = API: 손님(클라이언트)의 주문을 받아 주방(서버)에 전달하고 결과를 가져다 준다.

클라이언트 --[Request]--> API --[처리]--> 서버 (DB) <--[Response]-- <--[결과]--

Group C

동기 vs 비동기

동기 (Sync) = 은행 창구. 번호표 받고 내 차례까지 꼼짝 못함. 순서 보장. 블로킹.
비동기 (Async) = 카페 키오스크. 번호표 받고 자유. 음료 나오면 픽업.

async function fetchData() { try { const res = await fetch('/api/data'); // 기다리는 동안 다른 일 가능 const json = await res.json(); return json; } catch (err) { console.error(err); // 에러 처리 } }

Group D

JSON vs XML

// JSON (현대 표준 — 간결) { "student": "Kenny", "chapter": 2, "topics": ["API", "async", "JSON"] } // XML (옛 표준 — verbose) <student> <name>Kenny</name> <chapter>2</chapter> </student>

JSON = JavaScript Object Notation. key-value 쌍. 배열 지원. 사람도 읽기 쉬움. jarvis 응답 = JSON 구조.
