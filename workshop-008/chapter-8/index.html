Section A
클라우드 = 인터넷으로 서버 임대

클라우드 이전에는 온프레미스(On-Premise)로 물리 서버를 직접 구매·운영했다. 서버 용량 산정에 수개월이 걸리고, 초기 대규모 CapEx(자본 지출)가 필요했다. 클라우드는 가상화(Virtualization) 기술로 물리 서버를 여러 VM으로 분할해 필요한 만큼 임대한다. 5분 만에 서버 생성 가능, OpEx(운영 지출) 모델로 전환.

온프레미스 물리 서버 구매 → 수개월 설치 → CapEx 선투자 트래픽 예측 실패 시 → 유휴 or 다운 클라우드 콘솔/API → VM 즉시 생성 (5분) 트래픽 급증 → Auto Scaling 자동 대응 사용량 기반 과금 (OpEx)
Section B
IaaS — 인프라 임대

IaaS(Infrastructure as a Service)는 VM + 네트워크 + 스토리지를 임대한다. OS 설치부터 미들웨어, 런타임, 앱 배포까지 사용자가 직접 관리한다. 높은 자유도 대신 관리 부담이 크다. AWS EC2, Google Compute Engine이 대표적이다.

Section C
PaaS — 플랫폼 임대

PaaS(Platform as a Service)는 OS + 미들웨어 + 런타임이 이미 구성된 배포 플랫폼을 제공한다. 개발자는 코드와 앱 설정만 신경 쓰면 된다. 인프라 관리 부담이 대폭 감소한다. Vercel, Heroku, Google App Engine, Railway가 대표적이다. Newton 포탈은 현재 GitHub Pages(정적 PaaS)로 운영 중이다.

Section D
SaaS — 완성 소프트웨어 구독

SaaS(Software as a Service)는 완성된 소프트웨어를 구독(월정액)으로 사용한다. 설치 없이 브라우저나 앱으로 즉시 사용 가능. 업데이트는 벤더가 자동으로 제공한다. Gmail, Notion, Slack, Claude.ai, Wave(회계), Immich(셀프호스트 대안)가 예시다.

Section E
Docker / Kubernetes / Serverless

Docker는 앱과 모든 의존성을 컨테이너(Container)로 패키징한다. "내 PC에서는 됐는데" 문제를 해결한다. Kubernetes(K8s)는 수백~수천 개의 컨테이너를 오케스트레이션(스케줄링, 복구, 확장)하는 플랫폼이다. Serverless(AWS Lambda 등)는 서버 관리 없이 함수 단위로 실행하고 호출 횟수만큼 과금한다.

Quiz
  1. 온프레미스와 클라우드의 용량 산정(Capacity Planning) 접근 방식 차이를 설명하고, 클라우드에서 Auto Scaling이 왜 가능한지 말하라.
  2. IaaS, PaaS, SaaS에서 각각 사용자가 관리해야 하는 영역을 정리하고, 스타트업이 PaaS를 선택하는 이유를 설명하라.
  3. NewtonHQ가 현재 사용하는 IaaS, PaaS, SaaS 각 예시를 최소 1개씩 들어라.
  4. Docker 컨테이너와 VM의 차이를 설명하고, "내 PC에서는 됐는데" 문제를 Docker가 어떻게 해결하는지 말하라.
  5. Serverless 아키텍처의 장점과 콜드 스타트(Cold Start)라는 단점을 설명하라.
