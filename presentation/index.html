마음을 기억하는 방법

Kenny 의 시스템 소개  ·  2026-05-16

✎ “각 사람의 기억망 깨우치게 돕는다.”
— Newton School 의 한 문장

I. 감사의 말씀

✎선생님께 — 그동안 정말 감사했습니다.

✎그동안 소연이가 — 자기 마음을 터놓고 이야기할 수 있었던 유일한 사람이 선생님이셨습니다.

✎아이한테 — 너무나 큰 힘이 되었습니다.
부모로서 — 그 자리를 만들어 주신 것에 — 진심으로 감사드립니다.

✎오늘 이 자리에서 — 제가 요즘 하고 있는 일을 잠깐 나누고 싶었습니다.

✎선생님이 보시는 마음 의 결과 — 제가 만들고 있는 기억의 구조 가 어쩌면 — 가까운 자리에 있을 수도 있겠다는 생각이 들어서요.

II. Kenny 의 학습 — 어디서 출발해서 어디까지 왔나

✎제가 AI 를 제대로 처음 접한 게 — 약 10 개월 전입니다.

✎ 처음엔 — 정말 아무것도 몰랐습니다. YouTube 부터 시작했습니다. 영상 하나 보고, 따라 하나 만들어 보고, 부수고, 또 다른 영상 보고, 다시 만들고, 다시 부수고 — 그 반복이었습니다. 몇 달 그러다 보니 — 어렴풋이 원리 가 보이기 시작했습니다.

✎저는 25 년째 수학과 물리를 가르치고 있고, Bergen County 에서 학원도 운영하고, 부동산도 함께 하고 있습니다.

✎그런데 — 가르치는 일 말고 한 가지에 깊이 빠져 있습니다.

✎ “AI 와 함께 사람의 기억 구조를 설계하고 가르치는 일”

✎학문적으로는 Cognitive Architect for Human-AI Symbiosis — “사람이 어떻게 생각하고 기억하는지 보면서, AI 와 사람이 함께 더 잘 생각할 수 있도록 돕는 사람” 입니다.

✎ 원리를 알아낸 것 같다는 느낌이 든 게 — 한 2 주 정도 됐습니다. 그 후로 제 시스템 위에 그 원리를 하나씩 적용하는 중입니다. 계속 — 진화 중입니다.

III. 우리 시스템 — 뇌 와 외부 메모리 한 그림

✎ 처음엔 — 제가 뭘 하는지도 몰랐습니다. YouTube 따라 만들면서 “어, 이게 되네” 하다가 — 어느 날 이런 생각이 들었습니다. “더 — 사람처럼 쓰려면 어떻게 해야 할까.” 거기서부터 — 여기까지 왔습니다.

✎뇌가 어떻게 작동하는지를 들여다보면서, 그 구조를 그대로 컴퓨터 위에 옮겼습니다.

✎아래 표는 뇌의 기능과, 제 시스템이 어떻게 짝을 이루는지 보여줍니다.

✎
뇌제 시스템컴퓨터 위 자리
감각 기억 (찰나)대화창 — 지금 이 순간프롬프트 (사용자가 방금 적은 말)
작업 기억 (7개±2)지금 어깨에 얹힌 책들단기 저장소 (cache · 현재 세션 · 색인 카드)
해마 — 잠들 때 깨어나는 사서밤마다 기록을 정리하는 일꾼정리 일꾼 (“오라클” — 알맞은 자리로 옮김)
의미 기억 (사실 / 룰)룰 카드 + 정책 카드중기 저장소 (mind 영역)
일화 기억 (내가 겪은 것)날짜별 일지중기 저장소 (mind 영역)
절차 기억 (몸에 밴 것)자동 일꾼 (반복 작업)도구 영역 (armory)
장기 피질 (깊이 박힌 것)참고 문서 + 보관함장기 저장소 (asset / armory / 보관함)
시냅스 가지치기오래된 기록 정리 (예정)정리 일꾼 의 두 번째 일
수면 중 통합정기 정리 주기정해진 시간마다 자동 돎
재응고화 (기억 = 재기록)카드 수정 이력산출물 — asset / mind / armory 안
감정 가중치감성 카드카드 한 장이 다른 모든 카드의 톤을 정함 (template)
연상 활성화참조 흐름 (물 흐르듯 이어짐)언어 모델 (AI 가 한 생각에서 다른 생각으로 잇는 손)

✎뇌가 하는 일을 하나씩 짚어보면 — 제 시스템이 그 짝을 이미 갖고 있습니다. 우연이 아닙니다. 사람처럼 쓰려고 만들었으니까요.

III-bis. 표 아래는 — 모든 것이 설계 입니다

✎ ┌──────────────────────────────────────────────────────────┐ │ │ │ 감각 기억 → 사용자가 적는 한 줄 │ │ 작업 기억 → 어깨에 얹힌 책들 (단기) │ │ 해마 → 밤마다 옮기는 일꾼 = "오라클" │ │ │ │ ─── 여기 까지가 사람의 본능 │ │ ─── 아래부터는 — 제가 손으로 설계 한 부분 │ │ │ │ • 어떤 카드를 어디로 옮길지 │ │ • 어떤 기록을 얼마나 오래 둘지 │ │ • 어떤 카드가 다른 카드의 톤 을 정할지 │ │ │ │ = 뇌의 자연스러운 흐름을 — 컴퓨터 위에 의도적 으로 박은 것 │ │ │ └──────────────────────────────────────────────────────────┘

IV. 어떻게 만들었나 — 배운 것들의 합

✎이걸 만들면서 — 정말 많은 분야가 한꺼번에 필요했습니다.

✎ • programming 코드 짜기 • networking 데이터가 어떻게 흐르나 (집 - 사무실 - 인터넷 - 다시 집) • math recursion (자기 자신을 부르는 구조) fractal (작은 패턴이 큰 모양 만듦) probability (불확실성 다루기) • accounting double-entry bookkeeping (한 데이터를 두 군데서 — 정확성 체크) • test agent 만들고 → 다시 검증 ("내가 만든 게 정말 맞나" 자동으로 확인) • ... 그리고 매일 새로운 분야가 — 한 가지씩 더 들어옵니다.

✎처음엔 — 이 모두가 따로따로 보였습니다.

✎그런데 한 가지 공통점 이 있더군요.

✎ ┌───────────────────────────────────────────────────────┐ │ │ │ "데이터가 어떻게 흐르고, │ │ 어디에 박히고, │ │ 어떻게 다시 꺼내지는가." │ │ │ │ 분야마다 다른 옷 을 입었을 뿐 — │ │ 본질은 같은 한 가지 흐름 이었습니다. │ │ │ └───────────────────────────────────────────────────────┘

✎뇌도 — 그 일을 합니다. 매일 매 순간. 저는 — 그걸 컴퓨터 위에 옮기는 일을 하고 있습니다.

✎ ╭───────────────────────────────────────────────╮ │ │ │ 미래에는 — 어떤 일을 하든 │ │ 모든 것에 관심을 가지고 탐구해서 │ │ 자기 것 을 만들어야 합니다. │ │ │ │ 한 분야만 깊은 시대는 — 지났습니다. │ │ │ ╰───────────────────────────────────────────────╯
✎ 그래서 — 요즘은 유전학 과 물리학 도 다시 들여다보고 있습니다.

V. 4일 Method — 자기 발견 흐름

✎저는 이 시스템을 학생들에게도 가르칩니다. 4일이면 한 바퀴를 돕니다.

✎ Day 1 — Hub 가장 중요한 한 가지 찾기 "내가 이 단원에서 진짜 기억해야 할 것 하나는?" Day 2 — Cascade 생각이 어떻게 이어지는지 그리기 "이 생각에서 저 생각은 어떻게 나왔지?" Day 3 — Category 5개로 정리하기 (사람 머리는 5개가 한계) "이 모든 걸 5개 서랍에 나눠넣으면?" Day 4 — AI as Mirror AI 를 답 받는 도구가 아니라 자기 생각 비춰보는 거울로 쓰기

✎4일이 끝나면 학생이 자기 손글씨로 네 문장을 들고 집에 갑니다. 그게 앞으로 모든 공부의 출발점이 됩니다.
